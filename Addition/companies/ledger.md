# 新合同台账 字段提取规则（台账侧·单一规则源）

> 用途：本文是「新合同台账」表（每份合同 1 行）字段抽取与标准化的**单一规则源**。
> 运行时由 `companies/ledger_kb.py` 读取本文各分段，注入 DeepSeek 的**两阶段** prompt：
> Stage1 事实抽取（只问“合同写了什么”）→ Stage2 标准化映射（据下面的字段知识库把事实映射成标准字段值）。
> **改台账规则只改本文**（`BEGIN/END` 标记内的内容）。邮政线路表规则见 `companies/youzheng.md`。
>
> 取值口径（贯穿全文）：税率/补偿比例写**小数**（9%→0.09），保证金按**万元**数值，日期 **YYYY-MM-DD**，是否类只用 **是/否**；
> **相对期限**（“自签订之日起 N 年”）只填「合同开始时间」+「合同期限」，「合同结束时间」留 null 交由代码层推算。
> 列映射（Excel 列 ↔ 字段 ↔ 抽不抽）的机器权威是 `field_map.yaml`；“事实→字段”的证据回填索引是 `ledger_kb.py` 的 `FACT_KEYS`（内部管道，非业务规则）。

## 一、Stage1 事实抽取规则（只读不译）

<!-- BEGIN:stage1_rules -->
========== 第一阶段任务：合同事实抽取（只读不译） ==========
本阶段只问"合同里**写了什么**"，绝不做业务系统的标准化映射（那是第二阶段的事）。

原则：
1. 只提取合同中**明确写出**的内容，不推断、不补行业默认知识。
2. 保留**原文表述**，不改写、不概括。
3. 每个事实附 evidence，引用合同中对应的**原文片段**（必须是 OCR 文本里真实出现的字串）。
4. 信息不存在 → value 写 "未提及"。
5. 宁多勿少，把相关事实都抽出来。

需要提取的事实清单（key 用中文即可）：
- 合同名称(标题)、合同编号
- 甲方_全称、甲方_法定代表人、甲方_地址、甲方_省份、甲方_城市
- 乙方_全称、乙方_法定代表人、乙方_注册地址
- 合同期限(原文，如"自签订之日起2年"或"自2022年9月29日至2024年9月28日")
- 签署日期_甲方、签署日期_乙方、签署状态(双方是否盖章/签字日期是否完整)
- 业务范围描述、运输方式、合同性质(是否框架协议/中标位次)
- 付款账期(原文精确表述，如"收到增值税专用发票之日起30日内")
- 纳税人_税务信息(纳税人类型/发票类型/若合同直接写了税率也抄出来)
- 履约保证金(原文，含金额与收取口径)
- 旺季补偿(是否有/时间/上浮规则/比例)
- 油价联动(是否有/基准油价/触发条件/公式)
- 疫情补贴(是否有/标准)
- 甲方通知联系人及联系方式与通知地址
- 争议解决方式

输出 JSON 对象：
{ "事实名": {"value": "事实内容", "evidence": "原文引用"}, ... }
不要任何解释文字、不要 markdown 代码块以外的内容。
<!-- END:stage1_rules -->

## 二、字段知识库（Stage2 标准化映射规则）

> 每个字段：定义 / 取值格式 / 允许的值 / 映射条件→取值 / 注意。Stage2 据此把 Stage1 的事实映射成标准字段值。

<!-- BEGIN:field_kb -->
### 业务类型
定义: 干线业务类别（内部缩写）。
允许的值: 一干、二干、三干、省干、市内、其他
  - 条件: 标题/业务条款含'一级干线'或'一干' → 取值: → 一干
  - 条件: 含'二级干线'/'二干' → 取值: → 二干
  - 条件: 描述省内干线运输 → 取值: → 省干
  - 条件: 描述市内配送 → 取值: → 市内
注意: 依标题或业务条款判断；判断不了 → null，不要默认填'一干'。

### 合同类型
定义: 供应商分级（主选/备选）。
允许的值: 主选、备选
  - 条件: 框架协议 + 第一中标人/主选 → 取值: → 主选
  - 条件: 备选中标 / 第二中标人 → 取值: → 备选
注意: 依中标位次或'主/备选'表述；无明确依据 → null，不默认。

### 省
定义: 客户（甲方）所在省份（去掉'省'字，如'安徽省'→'安徽'）。
  - 条件: 从甲方全称/地址取省名 → 取值: 如'安徽省'→'安徽'

### 市
定义: 客户（甲方）所在地级市；**市只由甲方实体名称判断，不看办公/通讯地址**。
  - 条件: 甲方是'XX省XX市分公司/中心'（如'江苏省南京市…'）→ 市=该地级市（→ 南京）
  - 条件: 甲方是**直辖市区县级分公司**'北京市/上海市/天津市/重庆市 + XX区/县分公司'（如'北京市大兴区分公司'）→ 市=该区县名去掉'区/县'字（→ 大兴）
  - 条件: 甲方是**直辖市市级**'XX市分公司/XX市邮区中心'（含直辖市名但**名里无区县**，如'上海市分公司'、'上海市邮区中心'）→ 市=直辖市名（→ 上海）
  - 条件: 省级合同（甲方是'XX省分公司'、通篇无具体市/区）→ 市='/'
注意: **绝不用办公地址里的'XX区'定市**——地址只是办公地点、不代表甲方行政层级（甲方为'上海市分公司'、地址在'上海市浦东新区龙东大道'的，市=上海，不是浦东新区）。直辖市区县级才填区县（'北京市大兴区分公司'→大兴）、市级填直辖市名（'上海市分公司'→上海）。去掉'市/区/县'后缀只留地名。（代码层另有确定性兜底按甲方名归一。）

### 客户名称
定义: 客户（甲方/发包方）企业全称，按合同原文提取。
注意: 从'发包方（甲方）：'后原文提取，写什么就是什么，不加/删字。

### 合同主体
定义: 我方签约主体，即合同的**乙方/承包方/承运方**企业全称（承接邮政运输业务的物流公司）。
  - 条件: 合同写'承包方（乙方）：XXX有限公司' → 取值: → XXX有限公司（照抄原文全称）
注意: **取乙方、不是甲方**（甲方是邮政客户，已在「客户名称」）。绝不要填固定的某家公司名（如'北京博华物流'）——每份合同的乙方可能不同（天津智猪网、江西君卓、博华物流…），必须据本合同的乙方原文填；事实里没有乙方全称 → null。

### 合同开始时间
定义: 合同生效/起算日期。
取值格式: YYYY-MM-DD
  - 条件: 合同写'自签订之日起N年'等相对期限 → 取值: 开始时间=签署日期（甲乙方不同取靠后者）
  - 条件: 合同写明固定起止日（自X年X月X日至…） → 取值: 开始时间=该起始日

### 合同结束时间
定义: 合同到期日期。
取值格式: YYYY-MM-DD（无固定结束日则留 null）
注意: ⚠️ 关键口径：合同**写明固定结束日**才填；若只写'自签订之日起N年/月'这类**相对期限**、没有印出固定结束日 → 此字段留 **null**，并把时长原文填到「合同期限」字段，由系统推算结束日（禁止你自己算日期）。

### 是否完成签订
定义: 双方是否已完成盖章签字。
允许的值: 是、否
  - 条件: 有双方盖章/签字页（即便一方签署日期空白） → 取值: → 是
  - 条件: 明显缺一方签章 → 取值: → 否
注意: 以签章页为准；甲方盖章 + 乙方盖章即视为已签订（个别合同乙方日期空白但有章，仍算 是）。

### 合同期限
定义: 合同有效期/服务期的**主时长原文**（仅相对期限时有值）。
注意: 仅当合同用相对期限（'自签订之日起2年'/'24个月'）而非固定结束日时才填；照抄原文，**不要**换算成日期。只填**主时长**（如'自签订之日起2年'），**不要**带上'至下一轮采购完成/最多不超过N个月'等**延期条款**（延期是后续可能性、不计入基础结束日）。若合同已写明固定结束日，则本字段留 null。

### 账期
定义: 甲方收到发票后的付款账期（天数整数）。
取值格式: 天数整数（'30日内'→30）
  - 条件: 事实含'收到发票之日起X日内'或'X日'付清 → 取值: 提取数字 X
注意: 只按合同原文天数填；非天数表述保留原文。不要凭经验改成别的天数。

### 保证金
定义: 履约保证金金额，单位**万元**。
取值格式: 万元数值，保留1位小数（合同单位是元则 ÷10000）
  - 条件: 合同写'保证金XXX元' → 取值: XXX ÷ 10000，得万元
  - 条件: 合同写'保证金XX万元' → 取值: 直接取 XX
  - 条件: 合同只写'按全年运费的10%收取'、无具体金额 → 取值: → null（留空）
注意: 目标单位是**万元**：'3178420元'→317.8；'20万元'→20。绝不要把元的数值原样填进来。省级合同常无保证金→null。

### 税率
定义: 增值税税率（小数）。
取值格式: 小数（9%→0.09）
允许的值: 0.09、0.06、0.13、0.03
  - 条件: 合同**直接写明**税率 → 取值: 按合同写的，换算成小数
  - 条件: 未写税率，但'一般纳税人'+运输服务 → 取值: → 0.09（货物运输服务法定税率）
  - 条件: 一般纳税人 + 仓储/快递服务 → 取值: → 0.06
  - 条件: 小规模纳税人 → 取值: → 0.03
注意: 合同常只写'一般纳税人'+'增值税专用发票'而不写具体税率；据业务类型推断时置信调中等(≤0.8)。

### 是否有旺季补偿
定义: 合同是否有旺季价格上浮机制。
允许的值: 是、否
  - 条件: 有旺季上浮条款 → 取值: → 是
  - 条件: 确认无 / 正文未涉及 → 取值: → 否

### 旺季补偿时间
定义: 旺季的具体时间段（保留原文完整描述）。
注意: 无旺季补偿则 null。

### 旺季补偿规则
定义: 旺季价格调整的具体规则（保留原文完整描述）。
注意: 无旺季补偿则 null。

### 补偿比例
定义: 旺季价格上浮比例（小数）。
取值格式: 小数（上浮10%→0.10；'不超过15%'→0.15）。原文是区间/多值则保留原文字符串
注意: 无旺季补偿则 null。

### 是否有油价联动
定义: 合同是否有油价-运费联动机制。
允许的值: 是、否
  - 条件: 有油价联动条款 → 取值: → 是
  - 条件: 确认无 / 正文未涉及 → 取值: → 否

### 油价基准
定义: 油价联动机制中的基准油价（元/升）。
取值格式: 数值（元/升）
注意: 只取合同写明的基准油价；无油价联动则 null。注意单位以合同为准（个别合同误写'元/吨'，照抄数值并在证据注明）。

### 是否有疫情补贴
定义: 合同是否有疫情/公共卫生相关补贴条款。
允许的值: 是、否
  - 条件: 有疫情/公共卫生补贴条款 → 取值: → 是
  - 条件: 确认无 / 正文未涉及 → 取值: → 否

### 补贴标准
定义: 疫情补贴的具体标准（保留原文）。
注意: 无疫情补贴则 null。

## 三、转换方法演示（只示范方法/口径，严禁照抄其中数字）

> 下面的数字（500000→50、8%→0.08 等）只示意**换算关系**，不是任何真实合同的答案。当前字段取值只能由本次【事实】决定。

<!-- BEGIN:fewshot -->
1. 【保证金（单位换算 / 无金额留空）】合同写'缴纳保证金 500000 元' → 500000÷10000 = 50（万元）；写'保证金30万元' → 30；只写'按全年运费的10%收取'、无具体金额 → null（留空，不要把比例当金额、更不要凭空写个数字）。
2. 【税率（直读优先，否则按业务推断）】合同直接写'税率9%' → 0.09；未写税率但'一般纳税人'+运输服务 → 0.09（推断，置信≤0.8）；一般纳税人+仓储/快递 → 0.06；小规模纳税人 → 0.03。
3. 【补偿比例（百分比→小数；每份不同，绝不照抄）】'上浮8%' → 0.08；'上浮不超过15%' → 0.15；区间/多值'10%/15%' → 保留原文字符串。比例每份合同都不一样，必须按本合同原文，绝不沿用其它合同/演示里的比例。
4. 【合同开始/结束/期限（相对期限 vs 固定结束日）】① 相对期限：合同写'自签订之日起N年'、无固定结束日 → 合同开始时间=签署日（甲乙不同取靠后者）、合同结束时间=null、合同期限填'自签订之日起N年'（**只填主时长**，不要带'至下一轮采购完成/最多不超过N个月'延期尾巴），结束日交系统推算。② 固定结束日：合同写'自A年A月A日至B年B月B日' → 合同开始=A日、合同结束=B日、合同期限=null。
5. 【业务类型 / 合同类型（按标题与中标条款）】标题含'一级干线' → 业务类型'一干'；'二级干线' → '二干'。框架协议+第一中标人/主选 → 合同类型'主选'；备选中标/第二中标人 → '备选'；无明确依据 → null（不默认）。
6. 【省 / 市（含省级合同与直辖市口径）】甲方'XX省XX市分公司/中心' → 省=XX、市=XX；甲方是'XX省分公司'且通篇无具体地级市（省级合同）→ 省=XX、**市='/'**；甲方是**直辖市的区县分公司**（如'北京市大兴区分公司'）→ 省=北京、**市=大兴**（取区县名、去'区/县'字，不要填'北京市'）。
9. 【合同主体（取乙方，每份不同，绝不固定）】合同'承包方（乙方）：天津智猪网网络科技有限公司' → 合同主体='天津智猪网网络科技有限公司'；'乙方：江西君卓物流有限公司' → '江西君卓物流有限公司'。取的是**乙方/承包方**（我方承运主体），不是甲方邮政；绝不照搬某个固定公司名。
7. 【是否完成签订】双方都有盖章（即便一方签署日期空白）→ '是'；明显缺一方签章 → '否'。
8. 【是否类条款（旺季 / 油价 / 疫情）】正文有该机制条款 → '是'，并抽出对应的时间/规则/比例/基准/标准；合同确认无或全文未涉及 → '否'，对应的时间/规则/比例/标准 → null。

## 四、防照抄铁律

<!-- BEGIN:anti_copy -->
【防照抄铁律——最重要】
下面的"转换方法演示"只示范**事实→标准值的转换方法/口径**（如 元÷10000 得万元、纳税人类型推税率、
百分比写成小数、相对期限→结束日留空）。里面的数字（500000→50、8%→0.08 等）只是说明换算关系，
**不是任何真实合同的答案**。
- 当前每个字段的取值**只能由本次提供的【事实】决定**；事实里没有的就填 null。
- 绝不照抄演示里的数字，也不要用"邮政合同一般填什么 / 上一份填了什么"来补。
- 只信本次事实，不迁移、不脑补。留空永远优于自信地填错。
