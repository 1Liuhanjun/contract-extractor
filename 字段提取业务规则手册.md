## 二、字段详细规则

---

### §1 登记日期

**处理**: Python 代码  
**规则**: 取当日系统日期，格式 `YYYY/MM/DD`  
```
standardized["登记日期"] = date.today().strftime("%Y/%m/%d")
```

---

### §2 项目名称/编号

**处理**: 不填，空白

---

### §3 客户分类

**提取方式**: LLM 映射  
**Stage 1**: 提取甲方全称  
**Stage 2 规则**:

| 条件 | 结果 |
|---|---|
| 甲方名称含 `中国邮政` 或 `邮政` | `邮政` |
| 甲方名称含 `京东` | `京东` |
| 其他 | `其他` |

---

### §4 业务类型

**提取方式**: LLM 映射  
**Stage 1**: 提取合同标题 + 业务条款  
**Stage 2 规则**:

| 条件 | 结果 |
|---|---|
| 标题/条款含 `一级干线` / `干线` / `一干` | `一干` |
| 描述省内运输 | `省干` |
| 描述市内配送 | `市内` |

---

### §5 合同类型 ⚠️ 最复杂，注意区分甲方/乙方

**提取方式**: LLM 双维度判断  
**Stage 1**: 提取两类信息
1. 合同中**乙方（承包方）**被列为第几中标人/第几成交人的描述
2. 开头线路表格中每条线路的"线路性质"列（正式/备用）

**⚠️ 关键约束 — 区分甲方和乙方**:
- 只提取主语是**乙方（承包方）**的中标描述
- 甲方（发包方，如"中国邮政"、"江苏省分公司"等）的采购描述**不用于**判断乙方类型
- 如原文是"江苏省分公司...采购项目第一成交人"，这句话的主语是甲方，跳过不用

**Stage 2 判定逻辑**:

| 维度 | 信号 | 含义 |
|---|---|---|
| 中标人 | "第一中标人"/"第一成交人(主供应商)" | 主选侧 = ✅ |
| 中标人 | "第二中标人"/"第二成交人" | 备选侧 = ✅ |
| 线路表 | "正式"/"主选" | 主选侧 = ✅ |
| 线路表 | "备用"/"备选" | 备选侧 = ✅ |

**最终结果**:

| 主选侧 | 备选侧 | 填写 |
|:---:|:---:|---|
| ✅ | ✅ | `主选/备选` |
| ✅ | ❌ | `主选` |
| ❌ | ✅ | `备选` |

**默认**: 两个维度都没找到 → 默认 `主选`（博华是唯一/第一个乙方）

**示例（青岛合同）**: 线路表有"正式"和"备用"两种 → `主选/备选`

---

### §6-7 省 / 市 ⚠️ 直辖市特殊处理

**提取方式**: LLM  
**Stage 1**: 从甲方名称或地址提取  

**普通省份**: 省填省名，市填市名。都不加后缀。
- "安徽省合肥市" → 省=`安徽`，市=`合肥`

**直辖市（北京/上海/天津/重庆）**: 省填城市名，市填区名。
- "北京市大兴区" → 省=`北京`，市=`大兴`
- "上海市浦东新区" → 省=`上海`，市=`浦东`
- "重庆市渝中区" → 省=`重庆`，市=`渝中`
- "天津市滨海新区" → 省=`天津`，市=`滨海`

**判断方法**: 如果甲方地址以"北京市"/"上海市"/"天津市"/"重庆市"开头 → 走直辖市逻辑。

---

### §8 客户名称

**提取方式**: LLM  
**Stage 1**: 从"发包方（甲方）："后提取原文  
**规则**: 照抄原文，不修改。如合同写"合肥邮区中心"就不加"局"字

---

### §9 合同主体

**提取方式**: LLM  
**Stage 1**: 从"承包方（乙方）："后提取原文  
**可能值**: 
- 北京博华物流有限公司（最常见）
- 天津智猪网网络科技有限公司（也有）
- 其他

**⚠️ 不硬编码，从合同实际提取**

### §10 合同编码

**处理**: Python 代码  
**规则**: `BHWL-YZ邮政-YYYYMMDDNNN`
```
today_str = date.today().strftime("%Y%m%d")
existing = [已存在的当日编号数]
seq = str(len(existing) + 1).zfill(3)
→ "BHWL-YZ邮政-20260602001"
```

---

### §11 合同名称

**处理**: Python 代码  
**Stage 1**: 提取第一条线路名称（从线路表格第一行）  
**拼接**: `{线路名}一干运输合同`

**示例**: 第一条线路是"青岛-贵阳" → `青岛-贵阳一干运输合同`

---

### §12-13 合同开始时间 / 合同结束时间 ⚠️ 优先级关键

**提取方式**: LLM，有严格优先级

**读取顺序**:
1. **先看合同中间**"合同期限/有效期"段落（通常第十六条）
   - 如果有明确的起止日期（如"2022年10月5日至2023年3月31日"）→ **以此为准**
   - 如果该段落横线空白/无日期 → 进入第2步
2. **再看合同末尾**签章页的日期（如"2022年10月8日"）
   - 取**最后一个**签章日期

**关键原则**: `两处都有 = 只看中间，不看末尾`

**格式**: `YYYY-MM-DD`

**示例（青岛合同）**:
- 中间写了 `2022年10月5日至2023年3月31日`  
- 开始 = `2022-10-05`，结束 = `2023-03-31`

---

### §14-15 是否完成签订 / 是否同步财务

**处理**: 不填，空白

---

### §16 合同预警提醒

**处理**: Python 代码  
**规则**: `合同结束时间 - 今天 = 剩余天数`
```
days_left = (end_date - today).days
→ 正数=未到期，负数=已过期
```

---

### §17 账期（天）⚠️ 加法

**提取方式**: LLM  
**Stage 1**: 分别提取两段原文:
1. **结算周期**: 找"结算周期"、"对账周期"、"每月X日前"、"每月1日至月末日"等
2. **交付周期**: 找"收到发票之日起X日内"、"X个工作日内付"等

**Stage 2 计算**: `账期 = 结算周期天数 + 交付周期天数`

**典型值**: 结算周期15天 + 交付周期30天 = **45天**

**原文示例**:
> 结算周期: "以每自然月为结算周期，即：每月1日至当月月末日。甲方于每月**15日**前将上月...对账单交与乙方核对" → 结算周期约15天  
> 交付周期: "甲方收到乙方开具的...增值税专用发票之日起**30日**内将相应邮路运费付至乙方指定账户内" → 30天  
> → 账期 = 15 + 30 = 45天

**约束**: 两个数必须分别从两段原文中得出，不能只取一个。数字不一定固定为15+30，要看原文。

---

### §18 保证金（万元）

**提取方式**: LLM  
**Stage 1**: 提取第十五条履约保证金条款原文  
**Stage 2 规则**:

| 原文写法 | 处理 |
|---|---|
| "保证金XXX元" | XXX ÷ 10000 → 转万元 |
| "保证金XX万元" | 直接取XX |
| "按全年运费10%收取"（无具体数字） | 留空 `"/"` |

**示例**:
- "缴纳保证金3178420元" → `317.8` (3178420÷10000)
- "缴纳保证金20万元" → `20`
- "按全年运费10%收取" → `"/"`

---

### §19 税率

**提取方式**: LLM  
**Stage 1**: 提取纳税人类型、发票类型、税率数字  
**Stage 2 规则**:

| 条件 | 结果 |
|---|---|
| 一般纳税人 + 运输服务 | `9%` |
| 一般纳税人 + 仓储/快递 | `6%` |
| 小规模纳税人 | `3%` |
| 合同直接写了税率 | 直接用 |

---

### §20-23 旺季补偿（四件套，成组）⚠️

**提取方式**: LLM  
**Stage 1**: 提取旺季上浮条款原文 + 条款前是否有勾选框

**关键判定 — "是否有旺季补偿"**:
| 情况 | 结果 |
|---|---|
| 有旺季条款，且**已生效**（打勾/未划掉/无方框） | `是` |
| 无旺季条款 | `否` |
| 有条款但**未打勾/被划掉** | `否` |

**级联规则**: 如果"是否有旺季补偿"= `否`，则后面三个字段（旺季补偿时间、旺季补偿规则、补偿比例）**都不填**（输出 `"/"`）。

**各字段规则**:
- 旺季补偿时间: 保留原文（仅时间部分）
- 旺季补偿规则: 保留原文（仅规则部分）
- 补偿比例: 上浮百分比，格式 XX.XX%

**示例（青岛合同）**:
- 条款已生效 → 是
- 时间: `11月1-5日、11月11-17日、腊月十二至腊月二十九、正月初六至正月初十，共计35天。`
- 规则: `旺季期间的邮路价格（正班车辆除外）在合同结算价基础上上浮8%`
- 比例: `8%`

---

### §24-25 油价联动（两件套，成组）

**提取方式**: LLM  
**Stage 1**: 提取油价-运费联动条款原文

**"是否有油价联动"**:
| 情况 | 结果 |
|---|---|
| 有油价联动条款 | `是` |
| 无 | `否` |

**"油价基准"规则**: **只填数字**，不管单位是"元/升"还是"元/吨"。
- "0号柴油价格 8.16 元/升" → `8.16`
- "0号柴油价格 8.12 元/吨" → `8.12`

**级联**: 如果"是否有油价联动"= `否`，则油价基准填 `"/"`。

---

### §26-27 疫情补贴（两件套，成组）⚠️

**提取方式**: LLM  
**Stage 1**: 提取疫情/公共卫生相关条款

**"是否有疫情补贴" — 判断标准放宽**:

| 情况 | 结果 |
|---|---|
| 有疫情**补贴**条款 | `是` |
| 有疫情导致的**价格调整**条款（如"疫情等不受控风险可临时启用旺季价运作"） | `是` |
| 完全没有相关条款 | `否` |

**"补贴标准" — 留空约束**:

| 情况 | 填什么 |
|---|---|
| 有明确金额/比例/公式（如"每车补偿1000元"） | 填具体数字 |
| 表述模糊（如"单独申请"、"经批准后"、"另行协商"） | `"/"` — 不编造 |

**示例**:
- 合同写"因疫情等不受控风险...可临时启用旺季价运作" → 是（算疫情相关条款）  
  但补贴标准是"经乙方申请，甲方批准后" → `"/"`（没有明确数字）

## 三、空白字段的填充规则

| 原始值 | 输出为 |
|---|---|
| `None` / `""` / `"null"` | `"/"` |
| Stage 2 未提取到的字段 | `"/"` |
| 不填字段（§28-32 等） | `""`（空字符串，不是"/"） |

**"/" 适用的字段**：保证金、疫情补贴、补贴标准、旺季四件套、油价两件套、账期

## 四、提取优先级总结

```
1. Python 代码生成（5个：登记日期、合同编码、合同名称、合同预警提醒、项目名称）

```

## 五、常见陷阱 & 注意事项

1. **合同主体不能硬编码** — 乙方不一定是博华，可能是天津智猪网等
2. **合同时间中间优先** — 中间有日期就不看末尾签章，两处都有只看中间
3. **账期是加法** — 不是提取单个数字，是结算周期+交付周期
4. **合同类型看线路表** — 线路性质列有"备用"字样 → 有备选身份
5. **旺季未打勾=不存在** — 条款前有方框没打勾，视为没有
6. **油价只填数字** — 不管单位是元/升还是元/吨
7. **疫情补贴 ≠ "补贴"字面** — 疫情导致的价格调整条款也算
8. **补贴标准不编造** — 模糊表述（"单独申请"）→ 留空
9. **省/市不加后缀 + 直辖市特殊** — "安徽"不是"安徽省"；直辖市(北京/上海/天津/重庆)省填城市名、市填区名
10. **空值用"/"** — 匹配业务人员填写习惯
